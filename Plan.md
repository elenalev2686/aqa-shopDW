# Планирование автоматизации
## 1. Перечень автоматизируемых сценариев     
#### - UI тесты

  - Позитивные:

	* Оплата активной дебетовой картой (валидные значения):
	- ввод валидных значений в поля: «номер карты» - только 16 цифр; «месяц» - только двузначные числа от 01 до 12; «год» - двузначные числа, начиная от текущего и не более чем на 6 лет отстоящие позднее него; «Владелец» - только латинские символы; «cvv» - только трехзначные числа.
	Проверка статуса в базах данных, ожидаемый результат – APPROVED.
	* Покупка в кредит по данным активной дебетовой карты (валидные значения):
- ввод валидных значений в поля: «номер карты» - только 16 цифр; «месяц» - только двузначные числа от 01 до 12; «год» - двузначные числа, начиная от текущего и не более чем на 6 лет отстоящие позднее него; «Владелец» - только латинские символы; «cvv» - только трехзначные числа.
Проверка статуса в базах данных, ожидаемый результат – APPROVED.
	
  Негативные:

1. Покупка дебетовой картой:
    - оставить поле «номер карты» пустым, ввод количества символов меньшее 16. Ожидаемый результат – появляется сообщение: Неверный формат. Ввод более 16 символов – поле далее не заполняется.
    - оплата активной дебетовой картой № 4444 4444 4444 4441 (невалидные данные):
	- ввод в поле «месяц»  - 00, 13, ожидаемый результат – появляется сообщение: Неверно указан срок действия карты. Ввод спецсимволов, букв, трехзначных чисел, ожидаемый результат – поле не заполняется. Ввод однозначных чисел, оставление поля пустым, ожидаемы результат – появляется сообщение: Неверный формат.
	-ввод в поле «год» значения меньше текущего – 22, ожидаемый результат – появляется сообщение: Истёк срок действия карты; более 6 лет отстоящий от текущей даты  – 31 (1 месяц ранее текущей даты и + 6 лет), оставить поле пустым. Ожидаемый результата – появляется сообщение: Неверно указан срок действия карты. Ввод спецсимволов, букв, трехзначных чисел, ожидаемый результат – поле не заполняется. Ввод однозначных чисел, ожидаемы результат – появляется сообщение: Неверный формат.
	- ввод в поле «владелец» - кириллических символов, спецсимволов, цифр. Ожидаемый результат - появляется сообщение о неправильном вводе данных. Оставить поле пустым, ожидаемый результата - появляется сообщения: Поле обязательно для заполнения.
	- ввод в поле «cvv» одно-, двузначного числа, оставление поля пустым. Ожидаемы результата – появляется сообщение: Неверный формат. Ввод спецсимволов и букв, ожидаемый результат – поле не заполняется. Ввод четырехзначного числа – поле не заполняется. 
При проверке статуса в базах данных карта должна быть отклонена (DECLINED).
- попытка покупки неактивной картой (№ 4444 4444 4444 4442) – при проверке статуса в базах данных, должна быть отклонена (DECLINED).

## 2. Перечень используемых инструментов
### Для написания тестов будет использован язык java, учитывая то, что само приложение написано на java, исходя из этого же были выбраны все остальные необходимые инструменты (а также исходя из имеющихся навыков).
•	IntelliJ IDEA (учитывая имеющийся опыт написания авто-тестов).

•	JUnit (учитывая имеющийся опыт написания авто-тестов).

•	Gradle (учитывая имеющийся опыт написания авто-тестов и удобство использования данной системы управления зависимостями).

•	Selenide (учитывая имеющийся опыт написания авто-тестов, удобство в использовании данного фреймворка для автоматизированного тестирования веб-приложений).

•	Allure (учитывая его наглядность и удобство в использовании для создания подробных отчетов о выполнении тестов).

•	Docker (учитывая имеющийся опыт и возможности данной платформы для развертывания БД).

•	Git и Github (учитывая имеющийся опыт и возможности данной система контроля версий).
